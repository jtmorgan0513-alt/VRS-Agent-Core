REPLIT PROMPT — Fix Real-Time New Ticket Notifications for Agents and Admins
GOAL: New ticket notifications (sound, toast, browser notification, live data refresh) are not reaching agents or admins in real-time. Two bugs are causing this:

Agents in "working" status (actively working a claimed ticket) are excluded from new ticket broadcasts, so they stop hearing sounds after claiming their first ticket.
Admins/super_admins are completely excluded from new ticket broadcasts by role filter, and they have no online/offline toggle, so their status defaults to "offline" permanently.

CRITICAL: Do NOT modify the ticket submission flow, WebSocket connection/reconnection logic, notification sound/tone system, the agent online/offline toggle behavior, or any UI layout. The changes are strictly to the two server-side broadcast functions to fix who receives events.

Step 1 — Update broadcastToDivisionAgents in server/websocket.ts
Current code:
export function broadcastToDivisionAgents(division: string, event: WSEvent, excludeUserId?: number) {
  for (const [userId, sessions] of clients) {
    if (excludeUserId && userId === excludeUserId) continue;
    const primary = sessions[0];
    if (!primary || primary.role !== "vrs_agent") continue;
    if (primary.agentStatus !== "online") continue;
Replace the role check and status check with logic that:

Allows vrs_agent, admin, and super_admin roles through
For vrs_agent: allows both "online" and "working" statuses (not "offline")
For admin and super_admin: skips the status check entirely (they should always receive notifications when connected via WebSocket)

New code for the top of the function:
export function broadcastToDivisionAgents(division: string, event: WSEvent, excludeUserId?: number) {
  for (const [userId, sessions] of clients) {
    if (excludeUserId && userId === excludeUserId) continue;
    const primary = sessions[0];
    if (!primary) continue;
    
    const isAgent = primary.role === "vrs_agent";
    const isAdmin = primary.role === "admin" || primary.role === "super_admin";
    if (!isAgent && !isAdmin) continue;
    
    // Agents must be online or working; admins always receive when connected
    if (isAgent && primary.agentStatus !== "online" && primary.agentStatus !== "working") continue;
The rest of the function (the division matching and send logic) stays exactly the same. Admins already get all divisions assigned on WebSocket connect (line 77: divisions = Object.keys(DIVISION_LABELS)), so the existing division matching will naturally include them for all ticket types.

"Fix iPhone video playback issue — audio plays but video doesn't display:
Issue:

Videos uploaded from iPhone show audio only, no video image
iPhones record in HEVC/H.265 codec (MOV format) which has limited browser support

Solution — Convert videos to H.264/MP4 on upload:

Install FFmpeg:

Add FFmpeg to the server environment
This is the standard tool for video conversion


Convert uploaded videos:

When a video is uploaded, check if it's MOV/HEVC format
Convert to H.264 MP4 (universally supported):

bashffmpeg -i input.mov -c:v libx264 -c:a aac -movflags +faststart output.mp4

Store the converted MP4 version
Delete or archive the original MOV


Server-side conversion service:
javascriptconst ffmpeg = require('fluent-ffmpeg');

async function convertVideo(inputPath, outputPath) {
  return new Promise((resolve, reject) => {
    ffmpeg(inputPath)
      .outputOptions([
        '-c:v libx264',      // H.264 video codec
        '-c:a aac',          // AAC audio codec
        '-movflags +faststart', // Enable streaming
        '-preset fast',      // Balance speed/quality
        '-crf 23'            // Quality (lower = better, 23 is default)
      ])
      .output(outputPath)
      .on('end', () => resolve(outputPath))
      .on('error', (err) => reject(err))
      .run();
  });
}

Upload flow update:

Technician uploads video from iPhone
Server receives MOV/HEVC file
Server converts to MP4/H.264 in background
Store converted file in App Storage
Show "Processing video..." spinner until conversion complete
Save MP4 URL to submission


Accepted input formats:

MOV (iPhone native) — convert to MP4
MP4 — keep as-is if already H.264, convert if HEVC
WebM — keep as-is


Video player:

Use HTML5 <video> tag with MP4 source
Should now play in all browsers (Chrome, Safari, Edge, Firefox)


Fallback:

If conversion fails, keep original and show message: "Video may not play in all browsers. Download to view."
Provide download link as fallback



Install in Replit:

FFmpeg may need to be installed via Nix packages or as a binary
Alternative: Use a cloud video processing service (Cloudinary, AWS MediaConvert) if FFmpeg isn't available"

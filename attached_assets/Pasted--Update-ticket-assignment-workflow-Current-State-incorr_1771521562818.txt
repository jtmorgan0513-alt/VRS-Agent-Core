"Update ticket assignment workflow:
Current State (incorrect):

Tickets are auto-assigned to a specific agent on submission
Only that agent sees the ticket

New Workflow:
Stage 1 — Shared Division Queue:

Technician submits ticket with appliance type (maps to division)
Ticket goes into the division queue (NOT assigned to a specific agent)
assigned_to = NULL at this point
ALL agents assigned to that division can see and review the ticket
Example: A refrigeration ticket appears for all agents with refrigeration specialization

Stage 1 → Stage 2 Transition — Agent Takes Ownership:

When an agent clicks "Approve & Notify":

Ticket becomes assigned to that specific agent (assigned_to = agent_id)
SMS sent to technician
Ticket moves to Stage 2
Only the approving agent sees it in their Stage 2 queue



Stage 2 — Individual Agent Queue:

Agent who approved the ticket owns it
They are responsible for obtaining and sending the auth code
Ticket appears only in THEIR Stage 2 queue

Database Changes:

On submission: assigned_to = NULL, stage1_status = 'pending'
On Stage 1 approval: assigned_to = {agent_who_approved}, stage1_status = 'approved', stage1_reviewed_by = {agent_id}
On Stage 1 rejection: assigned_to = NULL (stays unassigned for resubmission)

Queue Views:
Stage 1 Queue (shared):

Show all tickets where:

stage1_status = 'pending'
appliance_type matches agent's division(s)
(OR agent is generalist)


Sort by: Oldest first (FIFO) or by urgency
Any qualified agent can click into and review

Stage 2 Queue (individual):

Show only tickets where:

stage1_status = 'approved'
stage2_status = 'pending'
assigned_to = {current_agent_id}


These are tickets THIS agent approved and owns

"My Assignments" Toggle:

In Stage 1: Toggle doesn't apply (queue is shared by division)
In Stage 2: Only shows agent's own tickets (already filtered this way)
Consider removing this toggle or repurposing it

Edge Cases:

Agent goes home mid-day with Stage 2 tickets:

Admin can reassign ticket to another agent
Add "Reassign" button (admin only) on Stage 2 tickets


Rejection:

Ticket stays in Stage 1 queue (unassigned)
If tech resubmits, it's a new ticket in the shared queue


Generalist agents:

See ALL division tickets in Stage 1
Can approve any ticket



UI Updates:
Stage 1 Queue Header:

"Stage 1 - Review ({count} pending)"
Shows total for agent's division(s)
No "assigned to me" filter needed

Stage 2 Queue Header:

"Stage 2 - My Authorizations ({count})"
Only shows this agent's approved tickets

Ticket Card in Stage 1:

Remove "Assigned to: {agent}" display
Show: SO#, Tech name, Appliance type, Time waiting, Urgency

Ticket Card in Stage 2:

Show: SO#, Tech name, Appliance type, "Approved by you at {time}""

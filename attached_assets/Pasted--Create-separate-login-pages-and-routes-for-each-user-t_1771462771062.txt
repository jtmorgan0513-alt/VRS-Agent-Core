"Create separate login pages and routes for each user type:
1. Three Separate Login Pages:
Field Technician Login — /tech/login

Clean, mobile-first design
Fields: LDAP ID only (no password)
Button: "Continue"
After LDAP lookup → proceed to service order entry → submission form
Link at bottom: "VRS Agent? Login here" (goes to /agent/login)

VRS Agent Login — /agent/login

Desktop-optimized design
Fields: LDAP ID + Password
Button: "Login"
After login → RGC code entry (if not entered today) → Stage 1 dashboard
Handle must_change_password flow
Link at bottom: "Admin? Login here" (goes to /admin/login)

Admin Login — /admin/login

Desktop-optimized design
Fields: LDAP ID + Password
Button: "Login"
After login → Admin dashboard
Handle must_change_password flow
No link to other logins (keep admin URL less visible)

2. Landing Page — / (root)

Simple page with two prominent buttons:

"Field Technician" → goes to /tech/login
"VRS Agent" → goes to /agent/login


Small link at bottom: "Administrator" → goes to /admin/login

3. URL Structure:
/                    → Landing page (choose user type)
/tech/login          → Technician LDAP entry
/tech/submit         → Submission form
/tech/history        → Technician's past submissions

/agent/login         → VRS Agent login
/agent/dashboard     → Stage 1 & 2 queues

/admin/login         → Admin login
/admin/dashboard     → Admin panel (users, analytics, settings)
4. Route Protection:

/tech/* routes — require valid LDAP ID in session
/agent/* routes — require authenticated VRS agent role
/admin/* routes — require authenticated admin role
Redirect to appropriate login if not authenticated

5. Session Handling:

Technician session: Store LDAP ID (no password, lighter session)
Agent/Admin session: Full JWT auth with role
Separate session handling so tech login doesn't interfere with agent login

6. Visual Differentiation:

Tech login: Mobile-friendly, simple, Sears blue
Agent login: Desktop, includes Sears logo, shows "VRS Agent Portal"
Admin login: Desktop, shows "VRS Administration"
Each page clearly labeled so users know they're in the right place"

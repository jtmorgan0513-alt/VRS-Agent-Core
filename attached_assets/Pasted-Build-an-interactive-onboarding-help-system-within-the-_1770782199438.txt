Build an interactive onboarding/help system within the app. Include:
1. First-Time User Wizard:

Detect if user has never logged in before (add first_login boolean to users table)
Show a step-by-step walkthrough modal on first login
Different flows based on role:

Technician: 5 slides covering Home dashboard → Submit form → Status tracking → Auth codes → History
VRS Agent: 5 slides covering Dashboard layout → Stage 1 queue → Approve/Reject → Stage 2 → Batch processing
Admin: 4 slides covering User management → Division assignments → Analytics → Settings


"Next", "Back", "Skip" buttons on each slide
Progress dots showing current step
Mark first_login = false when completed or skipped

2. Contextual Help Tooltips:

Add a "?" icon next to key UI elements
Clicking shows a tooltip with a brief explanation
Examples:

Request Type toggle: "Choose 'Authorization' for repairs over your limit, or 'Non-Repairable' if the unit can't be fixed"
Warranty Provider: "Select Sears Protect. B2B providers coming soon."
Stage 1 Approve button: "Confirms you have enough info to proceed. Tech will receive SMS and can leave the job site."



3. Help Center Page:

Accessible from sidebar/menu for all roles
Tabbed sections: Getting Started, How-To Guides, FAQs, Troubleshooting
Searchable content
Accordion-style expandable Q&A
Include all content from the user guide:

Technicians: Submitting requests, understanding status, receiving codes, resubmitting
VRS Agents: Reviewing submissions, approving/rejecting, sending auth codes, batch processing
Admins: Creating users, assigning divisions, viewing analytics



4. Restart Tutorial Option:

In user profile/settings, add "Restart Tutorial" button
Resets first_login = true so user can see the wizard again

5. What's New Modal (for future updates):

Add last_seen_version to users table
When app version changes, show a "What's New" modal highlighting new features
Store current version in environment variable

UI/UX:

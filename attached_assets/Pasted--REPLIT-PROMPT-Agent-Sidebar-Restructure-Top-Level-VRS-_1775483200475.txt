# REPLIT PROMPT — Agent Sidebar Restructure: Top-Level VRS / NLA Toggle

Restructure the agent dashboard sidebar to use a mode toggle instead of separate navigation sections. This is a UI-only change — no backend, API, or query logic changes needed. The existing `activeView` values, query params, stats, and WebSocket routing all remain exactly the same.

---

## THE PROBLEM

The current sidebar has the NLA navigation buried at the bottom as a separate section. Division Filters sit in the middle, making it unclear whether they apply to the standard queue or NLA. It's confusing — agents have to mentally map two different navigation structures.

## THE SOLUTION

One toggle at the top that switches the entire sidebar between VRS Authorization mode and NLA Parts mode. Same three nav items (Queue, My Tickets, Completed), same Division Filters — just operating on different ticket pools depending on the toggle.

---

### 4. Remove the old NLA Parts sidebar group entirely

The entire `{(agentDivisions.includes("nla") || isAdminViewing) && (...)}` block containing the NLA Parts group with NLA Queue, My NLA Tickets, and NLA Completed Today — DELETE IT. It's replaced by the toggle above.

### 5. Sync queueMode when activeView changes externally

If `activeView` is changed programmatically elsewhere in the component (e.g., after claiming a ticket, the code sets `setActiveView("nla_mytickets")`), the `queueMode` should stay in sync. Add a useEffect:

```tsx
useEffect(() => {
  if (activeView.startsWith("nla_")) {
    setQueueMode("nla");
  } else {
    setQueueMode("vrs");
  }
}, [activeView]);
```

This ensures the toggle always reflects the current view, even if the view changes from somewhere other than the toggle buttons.

### 6. Update header title

The header titles (around line 1119-1125) are already correct — they show different labels for standard vs NLA views. No changes needed.

### 7. Agents without NLA access

Agents who do NOT have the `"nla"` skill see NO toggle at all — just the standard Tickets, Division Filters, and Request Type groups. Their experience is completely unchanged. The toggle only appears when `hasNlaAccess` is true.

"Add AI-powered issue description enhancement as an experimental feature:
1. Environment Setup:

Add OPENAI_API_KEY to environment variables
Create /server/services/openai.js for API calls
Use gpt-4o-mini model (cost-effective for text)

2. Backend Endpoint:

POST /api/ai/enhance-description
Request body: { description: string, applianceType: string }
Sends to OpenAI with prompt:
You are a Sears appliance repair documentation assistant. Improve this technician's issue description for clarity and readability WITHOUT changing any of the content or meaning.

Your job is to:
- Break up long run-on sentences
- Organize the information in a logical, easy-to-scan format
- Fix spelling and grammar
- Elaborate slightly where helpful for clarity
- Keep all technical details, part numbers, and observations exactly as stated

Format the response as:
**Symptom:** [Customer's reported issue]
**Diagnosis:** [What the technician found]
**Repair Needed:** [Recommended parts/service]

Do NOT add information that wasn't in the original. Only clarify and reorganize.

Appliance Type: {applianceType}
Original Description: {description}

Return: { enhanced: string, original: string }
Rate limit: 5 requests per user per hour

3. Submission Form UI:

Below the Issue Description textarea, add:

"✨ Improve with AI" button
"🧪 Experimental" badge next to button (small, gray)


Button disabled until description has at least 30 characters
When clicked:

Button shows loading spinner with text "Improving..."
Disable textarea during processing
Call the enhance endpoint


On success:

Show enhanced version in a preview box below textarea
Preview box has light blue background with border
Label above preview: "AI-Improved Version (review and edit as needed)"
Two buttons below preview:

"Use This" — replaces textarea content with enhanced version
"Keep Original" — dismisses preview, keeps original text




After "Use This" is clicked:

Enhanced text goes into textarea
Textarea remains fully editable — technician can modify
Small note below textarea: "✨ AI-enhanced (edited)" or "✨ AI-enhanced" based on whether they made changes


On API error:

Show toast: "AI enhancement unavailable. You can still submit your original description."
Don't block submission



4. Experimental Feature Banner:

First time user clicks "Improve with AI", show a one-time tooltip/modal:

"🧪 Experimental Feature"
"This AI tool helps clarify your description without changing the meaning. Always review the result before submitting."
"Got it" button to dismiss
Store ai_tooltip_seen: true in localStorage so it only shows once



5. Database:

Add to submissions table:

original_description TEXT — Store what tech originally typed
ai_enhanced BOOLEAN DEFAULT false — Track if AI was used



6. VRS Agent View:

If submission has ai_enhanced = true:

Show small badge on description: "✨ AI-Enhanced"
Add "View original" toggle link
Clicking shows the original_description in a muted text box



7. Validation:

AI enhancement is optional — techs can always submit without using it
Minimum 30 characters to enable the button
Maximum 2000 characters for description (before and after enhancement)"

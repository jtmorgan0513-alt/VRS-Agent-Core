REPLIT PROMPT — WebSocket Notification Sound Fix for NLA + Admin Dashboards

Both functions currently have this line that skips non-agents:
if (!primary || primary.role !== "vrs_agent") continue;
Replace that check in BOTH functions so it allows vrs_agent, admin, and super_admin roles. The agentStatus !== "online" check should remain — only online users receive notifications. new_ticket → call playNotificationDing(), show toast "New [applianceLabel] ticket, [warrantyLabel]" with description SO #[serviceOrder], duration 8000ms, call showBrowserNotification with same title/desc, then invalidate admin queries. pending_tickets → show toast "Queued [applianceLabel] ticket, [warrantyLabel]" with description SO #[serviceOrder] waiting in queue, duration 8000ms, call showBrowserNotification, invalidate admin queries. No sound on pending_tickets (matches agent behavior — this is the existing queued-on-login notification, not a new incoming ticket).

Import playNotificationDing and showBrowserNotification at the top of admin-dashboard.tsx if they are not already imported. Check existing imports before adding.

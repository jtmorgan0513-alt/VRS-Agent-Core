## TASK: Show "X VRS Agents Online ┬À X Tickets in Queue" on Technician Home Screen

### What to build
A slim status banner on the technician home page (`tech-home.tsx`) that shows the real-time count of online VRS agents and queued tickets. Updates live via WebSocket whenever an agent goes online/offline or the queue count changes.

### Part 1: Backend ÔÇö New WebSocket helper function

In `server/websocket.ts`, add a new exported function:

```typescript
export function getOnlineAgentCount(): number {
  let count = 0;
  for (const [, client] of clients) {
    if (client.role === 'vrs_agent' && client.agentStatus === 'online') {
      count++;
    }
  }
  return count;
}
```

Also add a new broadcast function:

```typescript
export function broadcastToTechnicians(event: WSEvent) {
  for (const [, client] of clients) {
    if (client.role === 'technician') {
      sendToClient(client.ws, event);
    }
  }
}
```

### Part 2: Backend ÔÇö New API endpoint for initial load

In `server/routes.ts`, add a new PUBLIC endpoint (accessible to technicians):

```
GET /api/vrs-availability
```

Protected by `authenticateToken` only (no role restriction ÔÇö technicians need access).

This endpoint returns:
```json
{
  "onlineAgents": 3,
  "queuedTickets": 5
}
```

Implementation:
- `onlineAgents`: Call `getOnlineAgentCount()` from websocket.ts
- `queuedTickets`: Query `SELECT count(*) FROM submissions WHERE ticket_status = 'queued'`

### Part 3: Backend ÔÇö Broadcast availability to technicians on every change

In `server/routes.ts`, create a helper function near the top of the routes:

```typescript
async function broadcastVrsAvailability() {
  const onlineAgents = getOnlineAgentCount();
  const queuedTickets = await storage.getQueuedCountAll(); // new storage method, see below
  broadcastToTechnicians({
    type: 'vrs_availability',
    payload: { onlineAgents, queuedTickets }
  });
}
```

Add a simple storage method `getQueuedCountAll()` that returns the total count of submissions where `ticketStatus = 'queued'` with no division filter.

Call `broadcastVrsAvailability()` in these existing locations (ADD the call, do not remove or modify anything else):

Currently, the handler sets `divisions` for agents and admins. For technicians, `divisions` will be an empty array (they don't have specializations), which is fine ÔÇö they don't need division-scoped events.

After the existing `sendToClient(ws, { type: "connected", payload: { userId: decoded.id } });` line, add:

```typescript

if (client.role === 'technician') {
  const onlineAgents = getOnlineAgentCount();
  // We need queued count here ÔÇö either import storage or use a callback
  // Safest: make a simple count query
  try {
    const { storage } = await import('./storage');
    const queuedTickets = await storage.getQueuedCountAll();
    sendToClient(ws, {
      type: 'vrs_availability',
      payload: { onlineAgents, queuedTickets }
    });

      type: 'vrs_availability',

    });
  }
}
```
